1.  PROJECT OVERVIEW
================================================================================

BM Transcriber is a Bengali-focused audio transcription web application. Users
upload audio files, and the app sends them to an AI model (via OpenRouter) for
transcription. It supports two modes:
  - Basic transcription (plain text output)
  - Sync transcription (word-level timestamps + audio player with highlighting)

The app also has a full authentication system with IP-based account locking,
role-based admin panel, contact form, and in-memory session-based sync player.

4.1 HOME PAGE (index.html)
────────────────────────────
  - User opens http://127.0.0.1:5000
  - Page checks sessionStorage for a previous sync session ("Resume" banner)
  - User drags & drops an audio file (or clicks to browse)
  - Two buttons become enabled:
      A) "Transcribe" — basic text transcription
      B) "Transcribe & Sync" — word-level sync with audio player

4.2 BASIC TRANSCRIPTION FLOW
─────────────────────────────
  1. User clicks "Transcribe"
  2. Frontend POSTs to /transcribe (multipart/form-data with the audio file)
  3. Server (app.py:transcribe()):
     a. Resolves API key:
        - If user sends X-API-Key header → use that
        - Otherwise → fall back to server's OPENROUTER_API_KEY from .env
     b. Detects audio format from filename extension (.mp3, .wav, .ogg, etc.)
     c. Hardcoded model: google/gemini-3.1-flash-lite
     d. Size check:
        - If ≤ 25 MB → transcribe_inline() — base64 encode + single API call
        - If > 25 MB → transcribe_chunked() — split via ffmpeg in ~10 MB chunks,
          transcribe each segment, concatenate results
     e. Returns JSON: {"transcription": "..."}
  4. Frontend displays the transcribed text with a "Copy" button

4.3 SYNC TRANSCRIPTION FLOW (Transcribe & Sync)
─────────────────────────────────────────────────
  1. User clicks "Transcribe & Sync"
  2. Frontend POSTs to /transcribe-with-words (multipart/form-data)
  3. Server (app.py:transcribe_with_words()):
     a. Same API key resolution + format detection as basic flow
     b. Gets audio duration via ffprobe/ffmpeg (writes temp file, reads, deletes)
     c. Transcribes:
        - If ≤ 25 MB → transcribe_inline() + estimate_word_timestamps()
        - If > 25 MB → transcribe_chunked_with_ranges() + estimate_word_timestamps_chunked()
     d. Stores everything in IN-MEMORY dictionaries (SESSION_STORE, AUDIO_STORE)
        keyed by a random UUID (sync_session_id)
     e. Stores only the sync_session_id in Flask's session cookie
     f. Returns JSON:
        {
          "id": "...",        // sync_session_id
          "words": [...],     // [{word, start, end, index}, ...]
          "text": "...",      // full transcription text
          "duration": 123.4,  // seconds
          "audio_url": "/session-audio/<id>"
        }
  4. Frontend:
     a. Shows transcription preview
     b. Shows "Open Sync Player" button
     c. Saves session data to sessionStorage (survives browser navigation)
     d. User clicks → navigates to /player/<sync_session_id>

4.4 SYNC PLAYER PAGE (player.html)
───────────────────────────────────
  1. Page loads, triggers sync-player.js
  2. JS calls GET /api/session-transcriptions/<id> to fetch word data
  3. Audio URL: /session-audio/<id> (served directly from RAM)
  4. Renders each word as a clickable <span> with data-start and data-end attrs
  5. Features:
     - Real-time word highlighting via rAF (~60fps loop)
     - Click any word → seek audio to that position (with seek-guard)
     - Play/pause, seek bar, time display, speed control (0.5x-2x)
     - Auto-scroll active word into view
     - Edit transcript → save → server realigns timestamps (Needleman-Wunsch)
     - Reset to original
     - Copy transcript text
     - Keyboard shortcuts: Space (play/pause), ←/→ (rewind 5s / forward 5s)
     - Waveform-style progress bars
  6. Session recovery: if server data expired, falls back to sessionStorage

4.5 AUDIO CHUNKING (for files > 25 MB)
────────────────────────────────────────
  - ffmpeg is used to split the audio into ~10 MB chunks
  - Each chunk is transcribed individually via OpenRouter
  - Results are concatenated with segment markers: [Segment 1/3], [Segment 2/3], etc.
  - For sync player, each chunk gets its own time range, and words are distributed
    proportionally within that chunk's [start, end) range

  - No email verification required
  - Password minimum 8 characters
  - If email = "niabijoy123@gmail.com" → user gets admin role

5.2 LOGIN WITH IP LOCKING
──────────────────────────
  - GET/POST /login (rate-limited: 5 per minute)
  - On first successful login: user's IP address is stored in DB
  - On subsequent logins:
    - If IP matches → login succeeds
    - If IP is DIFFERENT → account is AUTOMATICALLY LOCKED
    - Admin must manually unlock via admin panel
  - Locked users cannot log in until unlocked

5.3 UNLOCK REQUEST FLOW
─────────────────────────
  1. Locked user visits /request-unlock
  2. Submits their email + a message
  3. An UnlockRequest record is created (status: "pending")
  4. Admin is notified via console print
  5. Admin can:
     - Approve → unlocks account, sets status to "approved"
     - Deny → sets status to "denied"
  6. User is notified via flash message on next login attempt

7.  IN-MEMORY SESSION SYSTEM (Sync Player)
================================================================================

  - Two global dictionaries in app.py:
    SESSION_STORE: sync_session_id → {words, text, duration, edited_text}
    AUDIO_STORE:   sync_session_id → {audio_bytes, audio_format, mime_type, timestamp}
  - Flask session cookie stores only the small sync_session_id string
  - Audio is served directly from RAM via /session-audio/<id>
  - Cleanup:
    - Soft TTL: 2 hours (sessions older than 2h are removed)
    - Hard TTL: 24 hours (if store exceeds 200 entries)
    - Cleanup runs every 2 hours on next /transcribe-with-words request
  - On server restart: all sessions are lost (client falls back to sessionStorage)

================================================================================
8.  SERVICES LAYER
================================================================================

8.1 TIMESTAMP ESTIMATOR (services/timestamp_estimator.py)
───────────────────────────────────────────────────────────
  - estimate_word_timestamps(text, duration_sec) → [{word, start, end, index}]
    Algorithm:
      1. Split text into tokens
      2. Assign each token a weight = character count
         - Sentence-ending punctuation (. ! ?) → weight × 1.5
         - Clause punctuation (, ; :) → weight × 1.3
      3. Distribute total duration proportionally by weight
      4. Last word clamped to exact duration
  - estimate_word_timestamps_chunked(full_text, segments, total_duration)
    Same algorithm but per-chunk (uses each chunk's [start, end) range)

8.2 WORD ALIGNER (services/word_aligner.py)
─────────────────────────────────────────────
  - align_edited_words(original_words, edited_text) → [{word, start, end, index, unchanged}]
  - Uses Needleman-Wunsch sequence alignment algorithm
    - Match: +2  |  Mismatch: -1  |  Gap penalty: -1
    - Case-insensitive, punctuation-insensitive fuzzy matching
  - Strategy:
    - Matched words → keep original timestamps
    - Inserted words → interpolate timestamps from nearest unchanged neighbors
    - Deleted words → removed from final array

      → Toolbar: "Edit", "Save" (hidden), "Cancel" (hidden), "Reset"
      → Scrollable transcript with word spans
      → Footer: word count + sync status indicator

  10.4 JavaScript Sync Engine (sync-player.js)
  ────────────────────────────────────────────────────────────────────────────
  Class/Module: IIFE (Immediately Invoked Function Expression)

  Key Functions:
    init()
      → Fetch transcription data from /api/transcriptions/<id>
      → Set up audio element
      → Call renderWords()
      → Wire all event listeners

    renderWords()
      → Loop through words array
      → Create <span> for each word with data-index, data-start, data-end
      → Add click handler for seekToWord()
      → Append to DOM with spaces between words

    findActiveWord(time)
      → Binary search on sorted words array
      → Returns index of word whose [start, end) contains currentTime
      → Gap filler: returns nearest word if between words

    onTimeUpdate()
      → Called via requestAnimationFrame throttle
      → Calls findActiveWord()
      → If word changed: call highlightWord() + scrollToWord()

    highlightWord(index)
      → Update all spans: add played/active/future classes

    scrollToWord(index)
      → element.scrollIntoView({ behavior: "smooth", block: "center" })

    seekToWord(index)
      → audio.currentTime = words[index].start

    enterEditMode() / exitEditMode()
      → Toggle contentEditable
      → Show/hide Save/Cancel buttons

    saveEdits()
      → Read innerText from transcript container
      → PUT to /api/transcriptions/<id>
      → Receive realigned words
      → Re-render word spans

    cancelEdit()
      → Restore original text
      → Re-render word spans

    resetTranscript()
      → Fetch original from API
      → Re-render

  Event Listeners:
    audio.timeupdate    → onTimeUpdate (throttled via rAF)
    audio.play          → update play button icon, status
    audio.pause         → update play button icon, status
    audio.ended         → update status to "Finished"
    audio.loadedmetadata → update time display
    seekBar.click       → seek to click ratio
    speedSelect.change  → audio.playbackRate = value
    playBtn.click       → toggle play/pause
    editBtn.click       → enterEditMode
    saveBtn.click       → saveEdits
    cancelBtn.click     → cancelEdit
    resetBtn.click      → resetTranscript
    document.keydown    → Space: play/pause, ArrowLeft/Right: ±5s

  11. ERROR HANDLING
================================================================================

  Scenario                           Handling
  ──────────────────────────────────────────────────────────────────────────
  No audio file provided             Return 400 error
  No API key configured              Return 500 error
  Transcription service fails        Return 502 error with message
  Audio file corrupted               Return 400 error
  FFmpeg not available for chunking  Return 400 error
  Transcription not found            Return 404, show error on player page
  Edited text empty                  Return 400 error
  Audio format unsupported           Browser handles natively
  Network error during transcribe    Frontend shows "Network error" message
  Rapid seeking                      Debounce via requestAnimationFrame
  Editing during playback            Sync continues, re-renders after save
